# GEMINI.md

Follow `CONTRIBUTING.md` for contribution policy and workflow.

Required flow for non-trivial changes:

1. issue (with milestone)
2. branch (`<actor>/<type>/<scope>/<task>-<id>`)
3. PR with valid title and linked issue
4. pass required checks
5. squash merge

Treat GitHub issue/milestone/release state as authoritative if docs drift.
